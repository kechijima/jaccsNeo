# FP業務管理ツール「jaccsNeo」 要件定義書

**バージョン:** 1.0  
**作成日:** 2026-04-10  
**ステータス:** 確定

---

## 1. システム概要

### 背景・目的

現行はkintoneで顧客管理・各種業務管理を行っており、**【個人・法人】パーソナルデータ（ID:173、約27,780件）** を中心に各サービスアプリへデータ連携している。本システムはこの構造を踏襲しつつ、FP間のコミュニケーション・ポータル機能・モバイル対応を強化した自社Webアプリ（jaccsNeo）に移行する。

### 対象ユーザー

FP担当者（社内利用のみ）。将来的に顧客向けポータルを追加予定。

### 技術スタック

| レイヤー | 技術 |
|---|---|
| フロントエンド | Nuxt 4 + Tailwind CSS |
| バックエンド / DB | Firebase Firestore |
| ホスティング | Firebase Hosting |
| 認証 | Firebase Authentication |
| ストレージ | Firebase Storage |
| 通知 | Firebase Cloud Messaging（FCM）+ メール（SendGrid） |
| リアルタイム | Firestore リアルタイムリスナー |

## 3. ユーザーロール・権限設計

### ロール定義

| ロール | 概要 | 対象 |
|---|---|---|
| **システム管理者** | 全データ・全設定の管理。JACCS等の制限コンテンツ管理 | システム管理者のみ |
| **理事会メンバー** | 全体の実績・活動データの閲覧。理事会スペースへのアクセス | 中央理事会メンバー |
| **EM2以上** | 所属グループ・組合の顧客データ管理。グループ横断の実績閲覧 | EM2・EM3・EM4・MM |
| **一般** | 担当顧客の管理・自組合スペースの閲覧・投稿 | 一般FP |
| **専門チーム** | 不動産/損保チーム専用データ・スペースへのアクセス追加付与（他ロールに兼務） | 不動産チーム・損保チーム |

> **専門チームについて:** 単独のロールではなく、他のロール（一般・EM2以上等）に対して追加付与するフラグとして管理する。例：「一般 ＋ 専門チーム（不動産）」

### 権限マトリクス

| 機能 | システム管理者 | 理事会メンバー | EM2以上 | 一般 | 専門チーム（追加） |
|---|---|---|---|---|---|
| 全顧客データ閲覧 | ✅ | ✅（閲覧のみ） | 自グループ | 担当のみ | 専門領域の案件 |
| 顧客データ編集 | ✅ | ❌ | 自グループ | 担当のみ | ❌ |
| JACCS制限コンテンツ | ✅ | ❌ | ❌ | ❌ | ❌ |
| チームダッシュボード（全体） | ✅ | ✅ | 自グループ | 自分 | — |
| スペース投稿（全体・公開） | ✅ | ✅ | ✅ | ✅ | ✅ |
| スペース投稿（非公開・役職限定） | ✅ | 理事会スペースのみ | EM限定スペース | ❌ | 専門チームスペース |
| ユーザー管理 | ✅ | ❌ | ❌ | ❌ | ❌ |
| CSVインポート（全顧客） | ✅ | ❌ | ✅ | ❌ | ❌ |
| CSVエクスポート（担当顧客のみ） | ✅ | ❌ | ✅ | ✅ | ✅ |
| グループ・組合マスタ管理 | ✅ | ❌ | ❌ | ❌ | ❌ |

```
/users/{uid}
  - name, email
  - role: "system_admin" | "board" | "em2_above" | "general"
  - specialTeams: string[]   # 例: ["real_estate", "non_life"] ※他ロールに追加付与
  - groupId, kumiaiId, position  # 所属グループ・組合・役職

/groups/{groupId}                         # Reterace / Asset / Miraito

/groups/{groupId}/kumiai/{kumiaiId}       # 各組合
  - name, adminId, members[]

  - type: "individual" | "corporate"
  - name, kana, birthdate, gender
  - contact: { phone, email, address, line }
  - occupation, income, employer
  - familyMembers[]

  - referredBy: customerId | null
  - status: "prospect" | "active" | "dormant" | "closed"

/customers/{customerId}/services/{serviceType}/cases/{caseId}
  - serviceType: "life_insurance" | "fire_insurance" | ...
  - status: "consulting" | "considering" | "contracted" | "completed" | "failed"

  - notes, attachments[]

/customers/{customerId}/referrals/{referralId}
  - referredCustomerId, date, serviceType

/spaces/{spaceId}
  - name, type, scope, groupId?, kumiaiId?
  - members[], admins[]
  - isPinned, isArchived

/spaces/{spaceId}/posts/{postId}
  - authorId, content, attachments[]
  - isPinned, reactions{}, createdAt

/spaces/{spaceId}/posts/{postId}/comments/{commentId}

/events/{eventId}
  - title, description, startAt, endAt, location
  - targetScope, attendees[]
  - createdBy, createdAt

/meetings/{meetingId}                     # 数字会議
  - spaceId, date, attendees[]

/restricted/{docId}                       # JACCS等の制限コンテンツ
  - title, content, attachments[]
  - accessLog[]: { uid, accessedAt }
```

## 5. 機能要件

### 5-1. 顧客管理（パーソナルデータ）

#### CSVインポート / エクスポート

**インポート（システム管理者・EM2以上）:**
- kintoneエクスポートCSVの形式に対応したインポート機能
- フィールドマッピング画面（CSV列 ↔ システムフィールドの対応設定）
- インポートプレビュー・バリデーション（重複チェック・必須項目確認）
- 差分インポート（既存データの上書き or スキップ選択）

**エクスポート（全ロール共通・担当顧客のみ）:**
- ログインユーザーが担当する顧客データのみエクスポート可能
- 出力形式: CSV / Excel
- エクスポート対象: 基本情報・家族構成・サービス対応状況
- 個人情報保護のため、他担当者の顧客データはエクスポート不可（Firestore Security Rulesで強制）

#### 顧客情報管理

- 個人・法人の区別（kintone ID:173/189 相当）
- 基本情報（氏名・フリガナ・生年月日・性別・連絡先・住所・LINE）
- 家族構成（続柄・年齢・職業・収入）
- 担当FP・所属グループ・所属組合
- 紹介元顧客の紐付け・紹介ネットワーク可視化
- 顧客ステータス管理

#### サービス対応状況（顧客ページ内一覧）

| カテゴリ | 管理するサービス |
|---|---|
| 保険 | 生命保険・火災保険・自動車保険（ソニー損保） |
| 不動産 | 購入・売却・賃貸（住居用/事務所用）・住宅ローン・家主 |
| キャリア | 転職（サークロス/アセット）・シニアプランニング |
| 通信 | 通信回線・通信環境診断・ピカラ光 |
| ライフ | 引越し・リフォーム・旅行・結婚式場紹介 |
| 法務 | 登記簿謄本・法人設立・遺言・相続・交通事故相談 |
| その他 | 地方紹介・HP制作・名刺制作・ウォーターサーバー 等 |

---

### 5-2. サービス案件管理

各サービス（保険・不動産等）ごとに案件を管理。パーソナルデータと紐付け。

- 案件番号・対応ステータス（相談中 / 検討中 / 成約 / 完了 / 不成立）
- 担当者・対応日時・成約金額（任意）
- 備考・添付ファイル（Firebase Storage）
- ワークフロー（ステータス進行管理）

---

### 5-3. ポータル機能（kintoneスペース相当）

#### スペース種別と公開範囲

| スペース種別 | 例 | 公開範囲 |
|---|---|---|
| 全体スペース | 全体連絡用・イベント告知・中央理事会 | 全FP |
| 活動報告スペース | Reterace/Asset/Miraito グループ活動報告・東西日本 | グループ全体 |
| 理事会・報告スペース | Reterace/Asset/Miraitoグループ理事会 | グループ管理者以上 |
| 組合スペース | りらくす組合・テゴリス組合 等 | 組合メンバーのみ |
| 数字会議スペース | 大阪水曜(岡)数字会議 等 | 担当者・参加者のみ |
| 専門チームスペース | 未来設計ハウジング・損保チーム | チームメンバーのみ |
| 役職スペース | EM活動報告・PM活動報告 | 該当役職者のみ |
| 地方事務所準備室 | 福岡・静岡・三重・京都・岐阜 | 準備室メンバーのみ |

#### 各スペースの機能

- 投稿フィード（テキスト・画像・ファイル添付）
- リアクション・コメント・メンション（@担当者）
- ピン留め（重要投稿を上部固定）
- メンバー管理（スペース管理者が追加/削除）

#### イベント告知

- イベント登録（タイトル・日時・場所・対象スペース指定）
- 参加申込・出席管理
- カレンダービュー（月・週・日）
- 通知（メール + アプリ内通知）

---

### 5-4. チームダッシュボード（必須）

#### 全体ダッシュボード（管理者向け）

- グループ別（Reterace/Asset/Miraito）の月次成約件数比較
- サービス別成約数ランキング（保険・不動産・転職 等）
- 新規顧客獲得数・紹介件数の推移グラフ
- アクティブFP数・活動件数

#### グループダッシュボード

- 組合別・FP別の活動数・成約数
- 数字会議の実績（週次・月次）

#### 個人ダッシュボード

- 今週のタスク・スケジュール
- 未対応案件アラート・フォローアップ要対応リスト
- 今月の活動サマリー・目標達成率

---

### 5-5. 制限コンテンツ管理（JACCS対応）

- 管理者のみアクセス可能なコンテンツ領域
- Firestore Security Rules でDB層から強制施行（UI制御だけでなくDB層で制御）
- 操作ログの記録（誰がいつアクセスしたか）

---

### 5-6. 通知（優先度順）

1. **アプリ内通知**（Firestore リアルタイム）
   - メンション・コメント・リアクション
   - 担当案件のステータス変更
   - イベント告知・リマインダー

2. **メール通知**（SendGrid）
   - イベント告知・リマインダー
   - 重要なお知らせ（管理者から全体）
   - 週次サマリー（任意設定）

## 6. 非機能要件

### モバイル対応（必須）

- スマホ・タブレット対応のレスポンシブデザイン
- タッチ操作最適化（大きめのボタン・スワイプ対応）
- PWA対応（ホーム画面追加・オフラインキャッシュ）

### セキュリティ

- Firebase Authentication によるログイン
- ロールベースアクセス制御（RBAC）
- Firestore Security Rules による細粒度アクセス制御
- 操作ログ・監査ログの記録
- セッションタイムアウト

### パフォーマンス

- Firestoreのインデックス設計による高速クエリ
- 顧客一覧は仮想スクロール対応
- 画像は Firebase Storage + CDN 配信

## 7. データ移行計画（kintone → jaccsNeo）

| ステップ | 内容 |
|---|---|
| 1. CSVエクスポート | kintoneから各アプリのデータをCSV出力 |
| 2. フィールドマッピング | kintone項目 ↔ jaccsNeo項目の対応表作成 |
| 3. パーソナルデータ移行 | 27,780件をバッチインポート（分割実行） |
| 4. 各サービスアプリ移行 | フェーズごとに順次インポート |
| 5. 並行運用 | kintone・jaccsNeo同時運用で検証 |
| 6. 切り替え | 確認後にkintoneを停止 |
